# نظام ولاء Classic Cafe ☕️🇯🇴

تطبيق ولاء رقمي فاخر لمقهى "Classic Cafe" — تسجيل دخول برقم الجوال، تجميع نقاط
(أكواب)، استبدالها بمكافآت، ولوحة كاشير محمية برمز PIN. البيانات الآن مخزّنة
على قاعدة بيانات حقيقية (Supabase) وتتزامن لحظياً بين جهاز الزبون وجهاز الكاشير.

## 1) التجهيز خطوة بخطوة

### أ) إنشاء قاعدة البيانات (Supabase — مجاني)
1. أنشئ حساباً على https://supabase.com وأنشئ مشروعاً جديداً (اختر أقرب منطقة، مثلاً Frankfurt).
2. من القائمة الجانبية: **SQL Editor** → **New query** → الصق محتوى ملف
   [`supabase-schema.sql`](./supabase-schema.sql) بالكامل → اضغط **Run**.
   هذا ينشئ جدولي الزبائن والحركات، ويفعّل التزامن اللحظي، ويضبط الصلاحيات.
3. من **Project Settings → API** انسخ:
   - **Project URL**
   - **anon public key**

### ب) ربط التطبيق بالقاعدة
1. انسخ `.env.example` وسمّه `.env.local`.
2. عبّي `VITE_SUPABASE_URL` و `VITE_SUPABASE_ANON_KEY` بالقيم من الخطوة السابقة.
3. غيّر `VITE_CASHIER_PIN` لرمز خاص بمقهاك (4 أرقام).

### ج) التشغيل محلياً
```bash
npm install
npm run dev
```
افتح الرابط الذي يظهر في الطرفية (عادة http://localhost:3000).

### د) النشر (لجعله مباشراً على الإنترنت)
أسهل طريقة عبر **Vercel** (مجاني):
1. ادفع الكود إلى GitHub (repo هذا نفسه).
2. من https://vercel.com اربط حسابك بـ GitHub → استورد الـ repo.
3. أضف نفس متغيرات `.env.local` (VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY, VITE_CASHIER_PIN)
   من إعدادات المشروع في Vercel: **Settings → Environment Variables**.
4. اضغط **Deploy**. بعد دقيقة يصير عندك رابط دائم تقدر تشاركه مع الزبائن وتطبعه كـ QR على الأكواب.

بديل: Netlify يعمل بنفس الطريقة تقريباً.

## 2) كيف يعمل التطبيق الآن

- **الزبون**: يدخل رقم جواله → إذا كان مسجلاً يدخل لبطاقته مباشرة، وإذا لم يكن
  مسجلاً يسجّل اسمه فيُنشأ له حساب فوراً في قاعدة البيانات.
- **الكاشير**: يفتح لوحة الكاشير برمز PIN، يبحث عن الزبون، ويضيف/يستبدل نقاطاً.
  أي تغيير يظهر فوراً على شاشة الزبون (حتى لو كانت مفتوحة على جهاز آخر) بفضل
  التزامن اللحظي (Realtime) من Supabase.
- **مسح QR**: الوضع الحالي هو **محاكاة** (الزبون يختار عدد النقاط ويضغط "مسح")
  لأن ربط قراءة QR فعلية بالكاميرا يحتاج طباعة أكواد QR حقيقية على الأكواب أولاً
  وتحديد آلية ربط كل كود برقم الزبون أو بقيمة ثابتة. إذا حبيت نفعّلها لاحقاً
  (قراءة كاميرا حقيقية لكود مطبوع)، هذا تطوير منفصل جاهزين نضيفه متى حددت شكل
  الأكواد المطلوبة.

## 3) ملاحظة أمان مهمة

رمز PIN الحالي محفوظ في متغيرات البيئة على جهة العميل (client-side) — وهو كافٍ
كخطوة أولى لحماية بسيطة من الزبائن العاديين، لكنه ليس حماية على مستوى إنتاجي
صارم (أي شخص يفحص كود الموقع بالمتصفح يقدر يجده). إذا أردت لاحقاً حماية أقوى
(تسجيل دخول حقيقي للكاشير بكلمة مرور)، نقدر نضيف Supabase Auth كخطوة تالية.
